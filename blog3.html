Fine-tuning Large Language Models on Custom Data: A Practical Walkthrough

Large language models are impressive out of the box, but they're not designed for your specific problem. A general-purpose LLM trained on web text won't understand your domain jargon, corporate policies, or customer service tone. This is where fine-tuning comes in.

Fine-tuning adapts a pre-trained model to your data without training from scratch. It's faster, cheaper, and produces models that respect your domain. In this post, I'll walk through a practical example: fine-tuning Mistral 7B on customer support dialogues.

When to Fine-tune

Fine-tuning isn't always necessary. Consider your alternatives:

  • Prompt engineering: Often solves the problem with no training. Try this first.
  • Retrieval augmentation (RAG): Better for knowledge-grounded tasks (Q&A, document search)
  • Fine-tuning: When you need the model to adopt a style, follow specific protocols, or handle domain-specific tasks

For customer support, you want consistency and domain knowledge, so fine-tuning is justified.

Preparing Your Data

Quality over quantity. A few hundred high-quality examples beat thousands of mediocre ones. Your training data should be representative of your use case.

Format matters. Most modern frameworks use a simple JSON or conversational format: 

{
  "messages": [
    {"role": "user", "content": "I can't reset my password"},
    {"role": "assistant", "content": "I can help with that. Let me verify your account..."}
  ]
}

Create a dataset of 200-1000 examples. Split 80/20 into training and validation. Use validation loss to detect overfitting.

Fine-tuning with Hugging Face

The transformers library with LoRA (Low-Rank Adaptation) makes this accessible. LoRA freezes the base model and trains a small adapter—drastically reducing memory and computation. 

from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments, Trainer
from peft import get_peft_model, LoraConfig, TaskType
from datasets import load_dataset

# Load model and tokenizer
model_name = "mistralai/Mistral-7B"
model = AutoModelForCausalLM.from_pretrained(model_name, device_map="auto")
tokenizer = AutoTokenizer.from_pretrained(model_name)

# LoRA configuration
lora_config = LoraConfig(
    r=8,
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    bias="none",
    task_type=TaskType.CAUSAL_LM
)

# Apply LoRA
model = get_peft_model(model, lora_config)

# Load your dataset
dataset = load_dataset("json", data_files="data.json")
dataset = dataset.map(lambda x: tokenizer(x["text"], truncation=True), batched=True)

# Training arguments
training_args = TrainingArguments(
    output_dir="./mistral-support",
    num_train_epochs=3,
    per_device_train_batch_size=4,
    save_steps=10,
    logging_steps=10,
    learning_rate=2e-4,
    warmup_steps=100,
)

# Train
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset["train"],
    eval_dataset=dataset["test"],
)
trainer.train()

Memory Optimization

Even with LoRA, fine-tuning can be memory-intensive. Use these tricks:

  • Quantization: Load the model in 8-bit or 4-bit format (load_in_8bit=True)
  • Gradient checkpointing: Trade compute for memory (gradient_checkpointing=True)
  • Smaller batch size: 4 or even 1 per device
  • Smaller models: Start with Mistral 7B or Llama 2 7B, not 70B

Evaluation & Iteration

Don't just look at loss. Evaluate on your validation set qualitatively. Generate outputs and ask:

  • Does the response follow your desired tone?
  • Does it stay in-character (avoiding generic or off-topic answers)?
  • Are domain terms handled correctly?

If performance is poor, consider:

  • Collecting more training data
  • Adjusting hyperparameters (learning rate, LoRA rank)
  • Using a different base model
  • Combining fine-tuning with RAG for external knowledge

Deployment Considerations

Once satisfied with your fine-tuned model, save it and prepare for deployment: 

# Save LoRA weights (much smaller than full model)
model.save_pretrained("./my-fine-tuned-lora")

# For inference, merge LoRA and base model
from peft import AutoPeftModelForCausalLM
model = AutoPeftModelForCausalLM.from_pretrained("./my-fine-tuned-lora")
merged = model.merge_and_unload()
merged.save_pretrained("./final-model")

Deploy via text-generation-webui, vLLM, or HuggingFace Inference API. Monitor outputs and collect feedback to iteratively improve.

Closing Thoughts

Fine-tuning bridges the gap between general-purpose LLMs and your specific needs. With modern tools and techniques, it's more accessible than ever. Start small, evaluate rigorously, and iterate. Your custom model will thank you.
